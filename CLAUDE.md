# CLAUDE.md — ナレッジベース作成ガイドライン

## 目的

本ナレッジベースは、**IT・AIの知識を持たない初心者が、独学でAIエンジニア兼データサイエンティストを目指すための資料集**として構築する。
Claudeとの対話を通じて得た知見を体系的に整理・蓄積し、学習の土台となるリファレンスを作ることをゴールとする。

---

## ドキュメント作成の原則

### 1. 読者は「IT・AI初心者」を想定する

すべての記述は、プログラミングやIT用語に不慣れな読者が**一人で読んで理解できること**を前提とする。

- 専門用語を使う場合は、必ずその場で平易な言葉で補足する
- 「当たり前」と思われる前提知識も省略しない
- 難しい概念ほど、身近なものに例えて説明する

### 2. 各用語・概念は以下の3点セットで記載する

| 項目 | 内容 |
|---|---|
| **ざっくり何か** | 一言で言うと何なのか。専門知識なしで伝わる説明 |
| **使い方・使用例** | 実際にどんな場面で使うのか。具体的なコマンドや操作も含める |
| **類似技術・関連用語** | 似たサービスや技術との比較。選び方の基準も添える |

### 3. たとえ話・アナロジーを積極的に使う

抽象的な概念は、日常生活の事物に置き換えて説明する。

**例：**
> ホスティングとは、家（Webサイト）を建てても土地（サーバー）がなければ人が訪問できないのと同じで、「土地を貸してくれる」サービスのこと。

### 4. 視覚的な表現を優先する

文章だけで説明するのではなく、以下を積極的に活用して直感的に伝える。

| 表現手法 | 使いどころ |
|---|---|
| **表** | 機能比較・料金比較・選択肢の整理 |
| **フロー図（テキスト形式）** | 処理の順序・データの流れ・システム間の連携 |
| **コードブロック** | コマンド・設定ファイル・具体的な操作手順 |
| **箇条書き** | 条件・手順・チェックリスト |

## ファイル構成

```
Study_Growing/
├── CLAUDE.md                        ← 本ファイル（作成ガイドライン）
└── AI_study/
    ├── README.md                    ← 全体インデックス・学習ロードマップ
    ├── AI.md                        ← AIサービス・LLM・MCP
    ├── Webapp.md                    ← Webアプリ開発
    ├── IT_Basic.md                  ← IT基礎・データ分析
    ├── インフラ.md                   ← クラウド・CI/CD・開発環境
    └── AI駆動開発.md                 ← AI×開発の手法・方法論
```

新しい用語や概念を追加する際は、上記のカテゴリのいずれかに分類して記載する。
どのカテゴリにも当てはまらない場合は、新しいファイルを作成してよい。

---

## カテゴリの分類基準

| カテゴリ | 収録する内容の例 |
|---|---|
| **AI** | LLM（大規模言語モデル）、AIサービス（Copilot・Gemini等）、MCP、ローカルAI実行 |
| **Webapp** | フロントエンド/バックエンド、Git、ホスティング、Vercel、Supabase、Firebase |
| **IT_Basic** | IT基礎用語、データ分析（Excel・Python）、ソフトウェア構造の概念 |
| **インフラ** | クラウド（AWS・GCP・Azure・Cloudflare）、CI/CD、Docker、ネットワーク、開発環境 |
| **AI駆動開発** | AIを使った開発手法、ハーネスエンジニアリング、ADR、チーム活用のパターン |

## 更新ルール

- Claudeとの会話で新しい用語・概念を学んだら、その都度該当カテゴリのファイルに追記する
- 既存の説明が古くなった場合は、上書きではなく「補足」として追記し、変更理由を残す
- 参考リンクは各ファイルの末尾にまとめて記載する
