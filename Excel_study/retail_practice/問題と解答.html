1
月別売上合計・前月比
SUMPRODUCT / MONTH
問題：2025年の月別売上合計を求め、2月以降の前月比（%）を算出せよ。
ヒント：MONTH関数で取引日から月を抽出し、SUMPRODUCT で条件集計する。
  • 集計表のB列（売上合計）を埋めよ。A列には 1〜12 の月番号が入っている前提。
  • C列（前月比）は 今月÷先月−1 で算出し、書式をパーセント表示にせよ。

数式

Excel Formula
' B2（1月売上）— A列に月番号が入っている前提 =SUMPRODUCT((MONTH('1_売上データ'!$B$2:$B$1001)=A2)*'1_売上データ'!$J$2:$J$1001) ' C3（2月以降の前月比） =IFERROR(B3/B2-1,"") ' → 書式：[赤]▲0.0%;[青]▽0.0%;— でわかりやすく

解答（数値）

なぜSUMIFSでなくSUMPRODUCT？
SUMIFS の条件範囲に MONTH(B:B) のような関数ネストはできない。SUMPRODUCT なら配列演算なので (MONTH(B:B)=A2) が使える。
※ ヘルパー列（=MONTH(B2) を作業列に展開）を作ってSUMIFSを使うのも高速でOK。

カテゴリ別 売上合計・構成比
SUMIF / 構成比
問題：6カテゴリ別に売上合計と全体に占める構成比（%）を求めよ。
ヒント：A列にカテゴリ名を入力し、SUMIF でカテゴリ条件の合計を出す。
  • B列：各カテゴリの売上合計
  • C列：全カテゴリ合計に対する構成比（書式をパーセントに）

数式

Excel Formula
' B2（カテゴリ別売上合計）— A列にカテゴリ名が入っている前提 =SUMIF('1_売上データ'!$F:$F, A2, '1_売上データ'!$J:$J) ' C2（構成比）— $B$2:$B$7 が全カテゴリの合計セル範囲 =B2/SUM($B$2:$B$7) ' または分母を固定セルに出しておく: =B2/$B$9 （B9 に SUM）

地域 × カテゴリ クロス集計
SUMIFS / 混合参照
問題：地域（行：東京〜札幌）× カテゴリ（列：家電〜書籍）のクロス集計表を作れ。
ポイント：混合参照を使い、数式を1つ入力してコピーだけで全セル埋まるようにせよ。
  • 行ヘッダ（A列）に地域名、列ヘッダ（1行目）にカテゴリ名を置く
  • B2 に数式を入力し、全体（B2:G6）にコピーして完成させる

数式（B2 に入力 → 全体にコピー）

Excel Formula
=SUMIFS('1_売上データ'!$J:$J, '1_売上データ'!$E:$E, $A2, ' ← 列を固定（地域） '1_売上データ'!$F:$F, B$1) ' ← 行を固定（カテゴリ） ' $A2 = 列$固定・行相対 → 横にコピーしても地域列はA列のまま ' B$1 = 列相対・行$固定 → 縦にコピーしてもカテゴリ行は1行目のまま

複合条件集計（5問）
SUMIFS / COUNTIFS / SUMPRODUCT
問題：次の5つの値を1セルの数式で求めよ。いずれも作業列は使わないこと。
  • ① ゴールド会員 かつ 東京 の売上合計
  • ② 家電 かつ クレジットカード の取引件数
  • ③ 2025年 Q2（4月〜6月）の売上合計
  • ④ 売上金額が 10,000円超 の取引件数
  • ⑤ 大阪 かつ ブロンズ会員 かつ 食品 の売上合計

数式と解答

Excel Formula
' ① ゴールド会員 × 東京 の売上合計 =SUMIFS('1_売上データ'!$J:$J, '1_売上データ'!$D:$D,"ゴールド", '1_売上データ'!$E:$E,"東京") ' → 3,893,540 ' ② 家電 × クレジットカード の件数 =COUNTIFS('1_売上データ'!$F:$F,"家電", '1_売上データ'!$K:$K,"クレジットカード") ' → 70 ' ③ Q2（4〜6月）売上合計 — 日付範囲はDATEで指定 =SUMIFS('1_売上データ'!$J:$J, '1_売上データ'!$B:$B,">="&DATE(2025,4,1), '1_売上データ'!$B:$B,"<"&DATE(2025,7,1)) ' → 14,388,940（SUMPRODUCT版: (MONTH>=4)*(MONTH<=6)*J でも可） ' ④ 売上金額 10,000円超 の件数 — 演算子は文字列で =COUNTIF('1_売上データ'!$J:$J, ">10000") ' → 454 ' ⑤ 大阪 × ブロンズ × 食品 の売上合計（3条件） =SUMIFS('1_売上データ'!$J:$J, '1_売上データ'!$E:$E,"大阪", '1_売上データ'!$D:$D,"ブロンズ", '1_売上データ'!$F:$F,"食品") ' → 32,090

解答一覧

問内容答え
①ゴールド会員 × 東京 の売上合計3,893,540 円
②家電 × クレジットカード の件数70 件
③Q2（4〜6月）の売上合計14,388,940 円

売上ランキング TOP 10
LARGE / INDEX+MATCH
問題：1〜10位の売上金額（LARGE）と、その金額の取引ID（INDEX+MATCH）を求めよ。
ヒント：A列に順位番号（1〜10）を入力し、B・C列に数式を入れてコピーする。

数式

Excel Formula
' B2（n位の売上金額）— A列に順位番号 =LARGE('1_売上データ'!$J:$J, A2) ' C2（その取引ID） =INDEX('1_売上データ'!$A:$A, MATCH(B2, '1_売上データ'!$J:$J, 0))

解答（数値）

順位売上金額取引ID（INDEX+MATCH）商品
1640,000TRX-00006ノートPC × 5個
2〜10640,000同上（※注意）ノートPC × 5個（同額）
⚠️ 重要な注意点：
TOP10 はすべて「ノートPC × 5個 = 640,000円」で同額のため、LARGE は全て 640,000 を返す。
MATCH は最初にヒットした行しか返さないため、C列の取引IDが全て同じになる（正確なランキング表にはならない）。
同額多数のケースには RANK + 補助列、または SORTBY（Excel365）を使うのがベター。
ボーナス問題：家電を除いた場合のTOP10はどうなるか？
=LARGE(IF('1_売上データ'!$F$2:$F$1001<>"家電",'1_売上データ'!$J$2:$J$1001),A2)
（Excel365なら通常入力可。旧バージョンは Ctrl+Shift+Enter で配列数式として入力）

統計分析
MEDIAN / STDEV / PERCENTILE / MAX / MIN
問題：売上金額列（J列）について、以下の統計量を求めよ。
  • ① 中央値（MEDIAN）
  • ② 標準偏差（STDEV）
  • ③ 90パーセンタイル（PERCENTILE）
  • ④ 最大値（MAX）/ 最小値（MIN）
  • ⑤ 四分位範囲 IQR = Q3（75%ile）− Q1（25%ile）

数式

Excel Formula
=MEDIAN('1_売上データ'!$J$2:$J$1001) ' ① 中央値 =STDEV('1_売上データ'!$J$2:$J$1001) ' ② 標本標準偏差（n-1） =PERCENTILE('1_売上データ'!$J$2:$J$1001, 0.9) ' ③ 90パーセンタイル =MAX('1_売上データ'!$J:$J) ' ④ 最大値 =MIN('1_売上データ'!$J$2:$J$1001) ' ④ 最小値（ヘッダ行除く） ' ⑤ IQR = Q3 - Q1 =PERCENTILE('1_売上データ'!$J$2:$J$1001,0.75) -PERCENTILE('1_売上データ'!$J$2:$J$1001,0.25)

解答（数値）

統計量値解釈
① 中央値（Median）7,500 円分布の中心。平均より外れ値の影響が小さい
② 標準偏差（StdDev）119,354 円平均からのばらつき幅。非常に大きい→高分散
③ 90パーセンタイル256,000 円上位10%の閾値。これ以上は要注目取引
④ 最大値（MAX）640,000 円ノートPC × 5個
④ 最小値（MIN）350 円歯ブラシ × 1個
⑤ IQR（Q3−Q1）45,000 円Q1=2,400 / Q3=47,400。中央50%のレンジ
分布の読み方：
中央値（7,500円）と標準偏差（119,354円）の乖離が大きい＝右裾が非常に長い分布。
IQR法での外れ値上限 = Q3 + 1.5×IQR = 47,400 + 67,500 = 114,900円。
この上限を超える取引件数：=COUNTIF(J:J,">114900") で確認してみよう。
STDEV vs STDEVP：
STDEV（標本標準偏差・n−1）はサンプルから母集団を推定する場合。
STDEVP（母標準偏差・n）はデータ全体が母集団である場合。今回は1,000件が全データのためSTDEVPも可。

自由課題
問題：以下のビジネス課題に対してどんな集計・分析を行うか、数式も含めて自分で設計せよ。

  • ④ 支払方法と会員ランクのクロス集計をして、ランク別の支払傾向を分析する
解答は1つではありません。どのアプローチを選ぶかが実務での「分析設計力」です。
