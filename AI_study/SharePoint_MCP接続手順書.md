# SharePoint × Claude MCP 接続手順書（初心者向け）

> Claude（Claude Code）から、社内SharePointのフォルダを自然言語で操作できるようにする手順書です。
> 「ターミナルって何？」というレベルの人でも、上から順にコピペすれば繋がるように書いています。

---

## 0. この手順でできること・できないこと

**できるようになること**

- Claudeに「あのサイトのフォルダのファイル一覧出して」と頼める
- ファイルの中身を読ませて要約させる、検索させる
- （権限があれば）ファイルのアップロードなど

**注意：会社のセキュリティ次第で「繋がらない」こともある**

社内SharePointは会社のルール（テナント設定）で守られています。手順どおりやっても、途中で「管理者の承認が必要です」と止まる場合があります。**それは失敗ではなく、会社のルール上の壁**です。その場合はIT部門への確認が必要になります（→ [トラブル対応](#7-うまくいかないときの確認リスト)）。

---

## 用語の超ざっくり説明

| 用語 | 一言でいうと |
|---|---|
| **MCP** | ClaudeとSharePointなどの外部サービスをつなぐ「接続規格」。USBの差込口みたいなもの |
| **MCPサーバー** | ClaudeとSharePointの間に立つ「通訳係」。Claudeの指示をMicrosoftの言葉に翻訳する |
| **ターミナル** | 文字でPCに命令する黒い画面。Macの「アプリ → ユーティリティ → ターミナル」 |
| **Node.js / npm** | MCPサーバーを動かすための土台ソフト。なければ入れる |
| **MFA** | パスワード＋スマホ承認などの本人確認。会社ログインで毎回出るアレ |

> たとえ話：MCPサーバーは**「外国（SharePoint）の窓口にいる通訳さん」**。あなた（Claude）が日本語で「この書類見せて」と言うと、通訳さんが現地の言葉で正しく頼んでくれます。

## 1. 事前準備：必要なものの確認

ターミナルを開いて、以下を1つずつ実行します。

### ① Node.js が入っているか

```bash
node -v
```

- `v18.0.0` など**数字が出ればOK** ✅
- `command not found` と出たら未インストール → 下の手順でインストール

**Node.jsのインストール（未インストールの人だけ）**

[公式サイト](https://nodejs.org/) から「LTS」版をダウンロードしてインストール。
または Homebrew が入っているなら：

```bash
brew install node
```

### ② Claude Code が入っているか

```bash
claude --version
```

- 数字が出ればOK ✅
- 出なければインストール：

```bash
npm install -g @anthropic-ai/claude-code
```

### ③ 自分のSharePointアクセス権

- ブラウザで対象のSharePointフォルダを**普通に開ける**ことを先に確認しておく
- 開けない場合は、そもそもアクセス権がない → 先にサイト管理者に依頼

✅ **チェックリスト**
- [ ] `node -v` で数字が出た
- [ ] `claude --version` で数字が出た
- [ ] ブラウザでSharePointフォルダを開けた

## 2. MCPサーバーをClaudeに登録する

ここでは初心者に一番やさしい **`@softeria/ms-365-mcp-server`** を使います。
（理由：会社ログインのMFAをそのまま通せる「デバイスコード方式」に対応しているため）

ターミナルで次の1行を実行：

```bash
claude mcp add ms365 -- npx -y @softeria/ms-365-mcp-server
```

| 部分 | 意味 |
|---|---|
| `claude mcp add` | ClaudeにMCPサーバーを登録するコマンド |
| `ms365` | 自分でつける「あだ名」。何でもいいが分かりやすく |
| `--` 以降 | 実際に動かす通訳係（MCPサーバー）の起動コマンド |

成功すると「Added MCP server ms365」のような表示が出ます。

---

## 3. Microsoftアカウントでログインする

登録しただけでは、まだ「鍵」がない状態です。次にログインして鍵を受け取ります。

### ① Claudeを起動

```bash
claude
```

### ② Claudeに「ログインして」と頼む

Claudeとの会話画面で、こう入力します：

```
ms365にログインして
```

すると、画面にこんな案内が出ます：

```
https://microsoft.com/devicelogin を開いて
コード「ABCD-EFGH」を入力してください
```

### ③ ブラウザでログイン

1. 表示された **URL**（`microsoft.com/devicelogin`）をブラウザで開く
2. 表示された **コード**（`ABCD-EFGH`など）を入力
3. **会社のアカウント** でログイン（メール → パスワード → スマホでMFA承認）
4. 「アプリにアクセスを許可しますか？」が出たら **許可**

> 💡 このとき「管理者の承認が必要」と出て進めない場合 → [トラブル対応](#7-うまくいかないときの確認リスト)へ

ログインが終わると、ターミナル側で自動的に「ログイン成功」と認識されます。鍵（トークン）はあなたのPC内に安全に保存され、毎回ログインし直す必要はありません。

---

## 4. ちゃんと繋がったか確認する

ターミナル（Claudeを一度終了して）で：

```bash
claude mcp list
```

`ms365` の横に **✓（チェック）** が付いていれば接続成功です 🎉

```
ms365   ✓ connected
```

## 5. 実際に使ってみる

Claudeを起動して、自然言語で頼むだけです。

**例1：ファイル一覧を見る**
```
Aidemy930サイトのShared Documentsフォルダのファイル一覧を見せて
```

**例2：ファイルを要約させる**
```
そのフォルダにある議事録のWordファイルを読んで、3行で要約して
```

**例3：検索**
```
このサイトから「予算」を含むファイルを探して
```

> Claudeが「どのサイト？」と聞き返してきたら、ブラウザのアドレスバーのURLを貼ると確実です。

## 6. 2回目以降の使い方

一度設定すれば、次からは**ただClaudeを起動するだけ**です。

```bash
claude
```

→ そのまま「○○のファイル見せて」と頼めばOK。
鍵の有効期限が切れた場合だけ、再度「ms365にログインして」を実行します。

## 7. うまくいかないときの確認リスト

| 症状 | 原因 | 対処 |
|---|---|---|
| `command not found: claude` | Claude Code未インストール | `npm install -g @anthropic-ai/claude-code` |
| `command not found: node` | Node.js未インストール | [nodejs.org](https://nodejs.org/) からLTS版を入れる |
| **「管理者の承認が必要」**で進めない | 会社が外部アプリを制限している | IT部門に「ms-365-mcp-serverを使いたい」と相談。自前のApp登録が必要な場合も |
| ログイン画面でMFAが通らない | 社用PC・社内ネット以外からの接続をブロック | 会社のPC・ネットワークから実行する |
| `claude mcp list` に出てこない | 登録コマンドが失敗していた | `claude mcp add` をもう一度実行 |
| 繋がるがファイルが見えない | そのフォルダへのアクセス権がない | ブラウザで開けるか確認。開けないなら権限申請 |

**MCPサーバーを消したいとき**

```bash
claude mcp remove ms365
```

## 8. もっと堅く使いたい場合（上級・参考）

会社が「デバイスコード方式」を許可してくれない場合は、IT部門と相談して**自前のApp登録**を作る方式になります。その際に必要になる主な情報：

| 項目 | 説明 |
|---|---|
| テナントID | 会社のM365の識別子 |
| クライアントID | 作成したアプリの識別子 |
| 権限（スコープ） | `Sites.Read.All` など。読むだけなら読み取り権限でOK |

この方式は管理者の承認（admin consent）が前提になります。詳細はIT部門の指示に従ってください。
